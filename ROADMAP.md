# Roadmap

This document outlines the planned direction for Saker. It is not a commitment
to specific timelines — priorities may shift based on community feedback and
available resources.

## Near-term (v0.x)

- [ ] Complete slog migration (21 files remaining)
- [ ] OpenAPI/Swagger specification for HTTP API
- [ ] Playwright E2E tests for web workspace UI
- [ ] Homebrew tap for macOS distribution
- [ ] SBOM and cosign signatures in release pipeline

## Mid-term (v1.0)

- [ ] Stable Go SDK API (guarantee backward compatibility)
- [ ] Multi-tenant project management (users/teams/projects)
- [ ] Plugin system for third-party tool and skill providers
- [ ] Kubernetes Helm chart for production deployment
- [ ] Comprehensive performance benchmarks and tuning guide

## Long-term

- [ ] Desktop app (Wails-based) for offline creative workflows
- [ ] Multi-agent orchestration with collaborative task delegation
- [ ] Video rendering pipeline with timeline-based editing API
- [ ] Community skill marketplace

## Contributing to the Roadmap

Open a GitHub Issue with the `enhancement` label to propose additions or
changes to this roadmap. Features that align with the KISS/YAGNI principles
and the existing modular architecture are more likely to be accepted.
